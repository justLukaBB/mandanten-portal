   - Use the same styling conventions as existing components (inline styles with rasolv CSS variables)

   - Simple component with "Einstellungen" heading and placeholder text
   - Reserved per user decision, not built in this phase

3. **main.tsx** — Wrap with BrowserRouter:
   - Import `BrowserRouter` from `react-router`
   - Wrap `<App />` in `<BrowserRouter>`
   - Keep existing `"./styles/index.css"` import

4. **App.tsx** — Replace useState-based view switching with Routes:
   - Import `Routes, Route, Navigate` from `react-router`
   - Remove the `useState<Client | null>` for selectedClient
   - Remove `mockClients` import (will be used directly in ClientList)
   - Define this route structure:
     ```
     <Routes>
       <Route path="/" element={<Navigate to="/clients" replace />} />
       <Route path="/dashboard" element={<DashboardPage />} />
       <Route path="/clients" element={<ClientListPage />} />
       <Route path="/clients/:id" element={<ClientDetailPage />} />
       <Route path="/settings" element={<SettingsPage />} />
       <Route path="*" element={<Navigate to="/clients" replace />} />
     </Routes>
     ```
   - Create a **ClientListPage** wrapper (inline in App.tsx or separate file) that:
     - Renders `<ClientList>` with `mockClients` data
     - Uses `useNavigate()` to navigate to `/clients/${client.id}` on row click instead of `setSelectedClient`
   - Create a **ClientDetailPage** wrapper that:
     - Gets `:id` from `useParams()`
     - Finds the client from `mockClients` by id
     - Renders `<ClientDetail>` with `onBack` using `useNavigate()` to go to `/clients`
     - Shows a "Client not found" message if id doesn't match any mock client
   - The layout stays: `<div className="flex h-screen overflow-hidden">` with `<Sidebar />` on the left and the routed content on the right via `<Routes>` inside the flex container

**URL scheme decision (Claude's discretion):** Using English route paths (`/clients`, `/dashboard`, `/settings`) because the API endpoints are English (`/api/admin/clients`) and mixing languages in URLs would be confusing. The UI labels remain German per the design.
