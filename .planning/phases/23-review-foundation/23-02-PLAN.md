**URL param pattern (from Phase 21):**
- Read all filter state from `useSearchParams`: page, limit, priority, search
- `setParam` helper that omits defaults from URL (page=1, limit=25, priority=all, search='') for clean URLs
- Local `searchInput` state for immediate typing feedback; 300ms debounce writes to URL param

**KPI Cards Section:**
3 cards using kpiStaggerContainer + kpiCardVariants for animation:

1. **Offen** — `data.total` (total open cases). Icon: Users. Color: accent blue.
2. **Hohe Prioritat** — count of clients where priority === 'high'. Compute client-side from `data.clients` response (since backend returns all for current page — for accurate count use `data.total` for "Offen" and client-side filter on current page data for "Hohe Prioritat"). Per CONTEXT.md Claude discretion: compute from the full current page of queue response data. Icon: AlertTriangle. Color: red.
3. **Durchschnitt Alter** — average of `days_since_payment` across current page clients. Show as "X Tage". Icon: Clock. Color: amber.

Each KPI value uses `useCountUp(value)` to animate from 0.
Show "vs gestern: +3" as static placeholder subtitle text per CONTEXT.md.
Card styling: white bg, 1px #E5E7EB border, 12px border-radius, no shadows (per design guidelines).

**Filter Bar:**
Row with:
1. Search input (left) — placeholder "Name oder Aktenzeichen suchen...", 300ms debounce, same pattern as client-list.tsx
2. Priority dropdown (right) — `<select>` with options: Alle, Hoch, Mittel, Niedrig. Maps to values: all, high, medium, low. Stored in URL params.

**Queue Table:**
Table with columns:
1. Avatar + Name — GradientAvatar with client name + name text (font-weight 600)
2. Aktenzeichen — client.aktenzeichen in JetBrains Mono font
3. Glaubiger — "X prufen" format showing documents_to_review count
4. Prioritat — colored pill badge: high (bg #FEF2F2, text #991B1B, border #FECACA), medium (bg #FFFBEB, text #92400E, border #FDE68A), low (bg #F0FDF4, text #166534, border #BBF7D0). Text: Hoch/Mittel/Niedrig. Per CONTEXT.md priority badge styling.
5. Alter — "X Tage" showing days_since_payment

Row hover: `bg-black/[0.02]` per CONTEXT.md.
Row click: `navigate(\`/review/${client.id}\`)`.

**Loading State:**
Skeleton rows (same pattern as client-list.tsx) — 5 skeleton rows with pulsing gray backgrounds.

**Empty State:**
Zero-results state with SearchX icon and "Keine Falle gefunden" headline + hint text + "Filter zurucksetzen" button (same pattern as client-list.tsx).

**Error State:**
If RTK Query returns error, show `toast.error('Review-Queue konnte nicht geladen werden')` on first load.

**Pagination:**
Bottom row with: "Seite X von Y" text, prev/next buttons (ChevronLeft/ChevronRight), disabled when at boundaries. Same pattern as client-list.tsx.

**Sort:**
Default sort: oldest first (days_since_payment descending) — this is already handled by the backend. No client-side sort needed for this phase.

**Priority filter:**
When priority !== 'all', filter the client-side response data. The backend returns all clients sorted by priority already, so client-side filtering is sufficient (same approach as flow filter in Phase 21).
