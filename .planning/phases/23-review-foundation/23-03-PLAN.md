**Layer 1: reviewApi.ts — Add search to ReviewQueueParams and URL params**

In `MandantenPortalDesign/src/store/api/reviewApi.ts`:

1. Add `search?: string` field to the `ReviewQueueParams` interface (after `priority`):
   ```typescript
   interface ReviewQueueParams {
     page?: number;
     limit?: number;
     priority?: ReviewPriority | 'all';
     search?: string;
   }
   ```

2. In the `query` function, destructure `search` from params and include it in the URL params (same conditional spread pattern as priority):
   ```typescript
   query: ({ page = 1, limit = 25, priority, search } = {}) => ({
     url: '/api/agent-review/available-clients',
     params: {
       page,
       limit,
       ...(priority && priority !== 'all' ? { priority } : {}),
       ...(search ? { search } : {}),
     },
   }),
   ```

**Layer 2: review-queue-page.tsx — Pass search to useGetReviewQueueQuery**

In `MandantenPortalDesign/src/app/components/review-queue-page.tsx`:

1. On line 232, change:
   ```typescript
   const { data, isLoading, error } = useGetReviewQueueQuery({ page, limit, priority });
   ```
   to:
   ```typescript
   const { data, isLoading, error } = useGetReviewQueueQuery({ page, limit, priority, search });
   ```

The `search` variable is already extracted from URL params on line 177 (`const search = searchParams.get('search') || ''`). It just needs to be passed into the query call. When `search` is an empty string, the reviewApi query builder will not include it in URL params (because the `...(search ? { search } : {})` spread evaluates to empty object for falsy/empty strings).

**Layer 3: agentReviewController.js — Implement server-side search filter**

In `server/controllers/agentReviewController.js`, inside the `getAvailableClients` method:

1. After the existing `priorityFilter` extraction (line 269), add search query param extraction:
   ```javascript
   const searchFilter = req.query.search ? req.query.search.toLowerCase().trim() : '';
   ```

2. After the priority filtering block (lines 384-387), add a search filter step. The search should filter on both `name` and `aktenzeichen` fields using case-insensitive substring matching. Insert AFTER the priority filter and BEFORE the pagination block (line 389):
   ```javascript
   // Filter by search term (name or aktenzeichen) if specified
   const searchedClients = searchFilter
       ? filteredClients.filter(c =>
           c.name.toLowerCase().includes(searchFilter) ||
           (c.aktenzeichen && c.aktenzeichen.toLowerCase().includes(searchFilter))
         )
       : filteredClients;
   ```

3. Update all references from `filteredClients` to `searchedClients` in the pagination block (lines 390-395):
   ```javascript
   const total = searchedClients.length;
   const pages = Math.max(Math.ceil(total / perPage), 1);
   const safePage = Math.min(page, pages);
   const start = (safePage - 1) * perPage;
   const end = start + perPage;
   const pagedClients = searchedClients.slice(start, end);
   ```

The ordering is: build availableClients -> sort -> filter by priority -> filter by search -> paginate. This ensures search and priority filters compose correctly and the total/pages reflect the filtered result set.
