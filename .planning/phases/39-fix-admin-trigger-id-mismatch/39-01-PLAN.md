<objective>
Fix the id/_id mismatch in secondLetterTriggerService.js that causes the admin manual trigger to silently fail.

Purpose: The admin route passes req.params.clientId which is the MongoDB ObjectId (_id). The service queries by { id: clientId } — the UUID id field — so findOneAndUpdate matches no document and returns null, which is interpreted as "alreadyTriggered". No state transition happens, no email is sent. This is the last open v10 requirement.

Output: Two query filter objects in secondLetterTriggerService.js changed from { id: clientId } to { _id: clientId }. REQUIREMENTS.md TRIG-02 checkbox updated to complete.
