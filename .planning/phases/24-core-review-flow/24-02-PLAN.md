Props:
```typescript
interface ReviewActionBarProps {
  clientId: string;
  creditor: ClientDetailCreditor;
  documentId?: string;
  formValues: Record<string, string>;
  originalValues: Record<string, string>;
  onActionComplete: (action: ReviewAction) => void;
  isLastCreditor: boolean;
  skipReasonsEnabled: boolean;
  onToggleSkipReasons: (enabled: boolean) => void;
}
```

Layout — fixed bottom bar inside the right panel (position sticky, bottom 0):

White bg, 1px top border #E5E7EB, padding 12px 16px. Flex row with gap.

Three buttons:
1. **Bestatigen** (green) — `bg-green-50 text-green-700 border border-green-200 hover:bg-green-100`. Icon: Check. Calls `useSaveReviewCorrectionMutation` with `{ clientId, body: { creditor_id: creditor.id, document_id: documentId, action: 'confirm' } }`. On success: dispatch `recordReviewAction({ creditorId, action: 'confirm' })`, call `onActionComplete('confirm')`.

2. **Korrigieren** (blue) — `bg-blue-50 text-blue-700 border border-blue-200 hover:bg-blue-100`. Icon: PencilLine. Builds corrections object from formValues (only fields that differ from originalValues to avoid sending unchanged data). Calls mutation with `action: 'correct'` and `corrections: { sender_name, sender_email, sender_address, reference_number, claim_amount, glaeubigervertreter_name, glaeubigervertreter_adresse, email_glaeubiger_vertreter }`. Per CONTEXT.md: "auto-saves all form changes and advances to next creditor in one step". Also sends `original` object with the original AI values for diff tracking. On success: dispatch `recordReviewAction({ creditorId, action: 'correct', corrections })`, call `onActionComplete('correct')`.

3. **Uberspringen** (orange) — `bg-orange-50 text-orange-700 border border-orange-200 hover:bg-orange-100`. Icon: SkipForward.
   - If `skipReasonsEnabled`: show inline skip reason selector before submitting
   - If `!skipReasonsEnabled`: immediately submit with action='skip' and advance

**Skip reason selector (inline, appears when Uberspringen clicked and skipReasonsEnabled):**
Renders below the action bar (or as a slide-up panel). Predefined categories:
- "Kein Glaubiger-Dokument"
- "Duplikat"
- "Unleserlich"
- "Unvollstandig"
- "Sonstiges" (with optional text input)

Small toggle at the bottom: "Begrudung deaktivieren" — flips `skipReasonsEnabled` via `onToggleSkipReasons`. Per CONTEXT.md: "when off, skip advances immediately without requiring a reason".

After selecting reason: calls mutation with `action: 'skip'`, dispatches `recordReviewAction({ creditorId, action: 'skip', skipReason })`, calls `onActionComplete('skip')`.

All three buttons show loading spinner during mutation. Disabled while mutation is in flight. Show error toast on failure via `toast.error()`.

Export: `export function ReviewActionBar()`

When `currentCreditorIndex` or review data changes, initialize `formValues` from the current creditor's data:
```typescript
useEffect(() => {
  if (currentCreditorWithDocs) {
    const c = currentCreditorWithDocs.creditor;
    const initial: Record<string, string> = {
      sender_name: c.sender_name || c.glaeubiger_name || '',
      reference_number: c.reference_number || '',
      sender_address: c.sender_address || c.glaeubiger_adresse || '',
      claim_amount: c.claim_amount != null ? String(c.claim_amount) : '',
      sender_email: c.sender_email || c.email_glaeubiger || '',
      glaeubigervertreter_name: c.glaeubigervertreter_name || '',
      glaeubigervertreter_adresse: c.glaeubigervertreter_adresse || '',
      aktenzeichen_glaeubigervertreter: c.aktenzeichen_glaeubigervertreter || '',
      email_glaeubiger_vertreter: c.email_glaeubiger_vertreter || '',
    };
    setFormValues(initial);
    originalValuesRef.current = { ...initial };
  }
}, [currentCreditorIndex, data]);
```
