**Refactor the `renderDocuments` function** in client-detail.tsx to use real `client.documents` data instead of the basic placeholder from Plan 22-01.

1. **Document count header:** Change `DOKUMENTE ({totalDocuments})` to use `client.documents?.length ?? 0`.

2. **Empty state:** If `client.documents` is empty or undefined, show:
   ```tsx
   <div style={{ padding: '40px 20px', textAlign: 'center', color: '#6B7280' }}>
     <div style={{ fontSize: '14px', fontWeight: 500 }}>Keine Dokumente vorhanden</div>
     <div style={{ fontSize: '12px', marginTop: '4px' }}>Für diesen Mandanten wurden noch keine Dokumente hochgeladen.</div>
   </div>
   ```

3. **Document list:** Iterate over `client.documents` and render each document with the prototype's visual design (already established in the mock version). For each document:

   - **Name:** `doc.name || doc.filename || 'Unbekanntes Dokument'`
   - **Checkmark icon:** Show green check if `doc.processing_status === 'completed'`, amber spinner/clock if `processing`, red X if `failed`
   - **Warning badge:** If `doc.manual_review_required === true`, show the amber badge: `● Manuelle Prüfung erforderlich` (same style as prototype)
   - **AI confidence score:** Use `doc.confidence ?? doc.ai_confidence ?? 0` (the backend stores it in either field). Display as:
     - Text: `AI-Sicherheit: {confidence}%` (keep the prototype's label)
     - Progress bar: Same style as prototype — 4px height, #E5E7EB background, #FF5001 fill, 200px width, fill width = `${confidence}%`
     - If confidence is 0 or undefined, show `AI-Sicherheit: –` with no progress bar
   - **Status text:** Derive from `doc.processing_status` and `doc.document_status`:
     - If `doc.document_status === 'creditor_confirmed'` → "Bestätigt" (green)
     - If `doc.manual_review_required` → "Prüfung ausstehend" (amber)
     - If `doc.processing_status === 'completed'` → "Verarbeitet" (green)
     - If `doc.processing_status === 'processing'` → "Wird verarbeitet..." (amber)
     - If `doc.processing_status === 'failed'` → "Fehlgeschlagen" (red)
     - Default: "Ausstehend" (gray)
   - **Download and Details buttons:** Keep the prototype's button layout. The Download button can link to `/api/admin/clients/${clientId}/documents/${doc.id}/download` (the backend has this endpoint). Details button can be a no-op for now (future feature).
   - **Upload date:** Show `doc.uploadedAt` formatted as German date below the document name if available

4. **Summary row at top (Recent Documents in Übersicht):** The overview tab's "LETZTE DOKUMENTE" card should also use real data — show the last 3 documents from `client.documents` sorted by uploadedAt descending. Use `client.documents.slice().sort((a, b) => new Date(b.uploadedAt).getTime() - new Date(a.uploadedAt).getTime()).slice(0, 3)`.

5. **Keep the "Alle herunterladen" button** but make it a no-op for now (or remove it — Claude's discretion). The backend doesn't have a bulk download endpoint.

6. **Preserve the prototype's visual design** — same border-radius, padding, font sizes, colors, spacing. Only the data source changes.
