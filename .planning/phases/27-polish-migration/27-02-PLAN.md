    MandantenPortalDesign/src/app/components/review-queue-page.tsx

  <action>
In `review-queue-page.tsx`, enable auto-refresh and highlight new cases:

**1. Enable polling on existing query hook:**
The `useGetAdminReviewQueueQuery` call already exists. Add `pollingInterval: 30000` to its options object. This makes RTK Query automatically re-fetch every 30 seconds.

Example:
```ts
const { data, isLoading, isFetching } = useGetAdminReviewQueueQuery(
  { page, limit, ...(priority !== 'all' ? { priority } : {}), ...(debouncedSearch ? { search: debouncedSearch } : {}) },
  { pollingInterval: 30000 }
);
```

**2. Track previous client IDs to detect new cases:**
Use a `useRef<Set<string>>` to store the set of client IDs from the previous fetch. After each successful data load, compare the new client IDs against the previous set. Any ID not in the previous set is a "new case."

Store the set of new IDs in a `useState<Set<string>>` called `highlightedIds`. After 3 seconds, clear the highlighted set (use a setTimeout in a useEffect).

**3. Apply highlight animation to new rows:**
For each queue row, check if its ID is in `highlightedIds`. If so, apply a subtle background highlight that fades out:
- Use framer-motion's `animate` prop: `backgroundColor: ['#FEF3C7', '#FFFFFF']` with `transition: { duration: 2 }` on the row `motion.div`.
- Alternatively, apply a CSS class with a keyframe animation (fadeHighlight: from amber-50 to transparent over 2 seconds).

Choose the framer-motion approach since motion.div rows are already in use.

**4. Show subtle fetch indicator:**
When `isFetching` is true (background re-fetch, not initial load), show a small subtle indicator — e.g., a thin animated bar at the top of the table or a small spinning icon near the header. Keep it unobtrusive since polling runs frequently. Do NOT replace the table content with skeleton rows during background fetches (only on initial load).

**Important:** Do NOT pause polling when the user is on a review detail page (/review/:clientId). The queue page component unmounts when navigating away, which naturally stops polling. When they return, polling resumes. This is the default RTK Query behavior — no special handling needed.
  </action>

    - After 30 seconds, data re-fetches (visible in Network tab or via isFetching indicator)
    - If a new case appears between refreshes, its row briefly highlights amber before fading to white
  </verify>
