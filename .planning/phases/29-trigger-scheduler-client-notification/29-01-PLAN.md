**Class: SecondLetterTriggerService**

**Constructor:** Accepts `{ emailService }` dependency (injected from server.js, matching how documentReminderService and loginReminderService receive dependencies).

**Method: triggerForClient(clientId, actor = 'system')**

1. Atomic `findOneAndUpdate` on Client model:
   - Filter: `{ id: clientId, second_letter_status: 'IDLE' }`
   - Update `$set`:
     - `second_letter_status: 'PENDING'`
     - `second_letter_triggered_at: new Date()`
     - `second_letter_form_token: uuidv4()`
     - `second_letter_form_token_expires_at: new Date(Date.now() + 14 * 24 * 60 * 60 * 1000)` (14 days)
   - Update `$push` on `status_history`:
     - `{ id: uuidv4(), status: 'second_letter_pending', changed_by: actor === 'system' ? 'system' : 'admin', metadata: { actor, reason: actor === 'system' ? 'auto_30_day_trigger' : 'admin_manual_trigger', triggered_at: new Date().toISOString() }, created_at: new Date() }`
   - Options: `{ new: true }`

2. If `findOneAndUpdate` returns `null` (client was NOT IDLE — idempotency guard):
   - Fetch current status with `Client.findOne({ id: clientId }, { second_letter_status: 1 })`
   - Return `{ success: false, alreadyTriggered: true, currentStatus: existing?.second_letter_status || 'unknown' }`

3. After successful transition:
   - Build portal URL: `${process.env.PORTAL_BASE_URL || 'https://mandanten-portal.onrender.com'}/second-letter?token=${client.second_letter_form_token}`
   - Call `this.emailService.sendSecondLetterNotification(client.email, \`${client.firstName} ${client.lastName}\`, portalUrl, client.aktenzeichen)`
   - Return `{ success: true, clientId: client.id, aktenzeichen: client.aktenzeichen, emailSent: emailResult.success, emailId: emailResult.emailId }`

4. Wrap entire method in try/catch. Log errors with `console.error('❌ Error triggering second letter for client:', clientId, error.message)`. Re-throw so caller can handle.

**Method: checkAndTriggerEligible()**

1. Calculate `thirtyDaysAgo = new Date(Date.now() - 30 * 24 * 60 * 60 * 1000)`

2. Query: `Client.find({ second_letter_status: 'IDLE', 'final_creditor_list': { $elemMatch: { email_sent_at: { $exists: true, $ne: null } } } }).select('id aktenzeichen firstName lastName email final_creditor_list')`
   - This gets all IDLE clients who have at least one creditor with email_sent_at set.

3. Filter in JavaScript (two-step approach per research — avoids complex aggregation):
   - For each client, extract all `email_sent_at` dates from `final_creditor_list`, filter nulls, compute `Math.max(...dates)`
   - Only include clients where `maxSentAt <= thirtyDaysAgo`
   - Skip clients where no creditor has `email_sent_at` set (first round was never sent)

4. Iterate over filtered clients sequentially (not parallel — avoid overwhelming Resend rate limits):
   - For each client, call `this.triggerForClient(client.id, 'system')`
   - Track counts: `triggered`, `skipped` (alreadyTriggered), `errors`
   - Catch per-client errors — log and continue, do not abort the batch

5. Return `{ triggered, skipped, errors, total: filtered.length }`

6. Log summary: `console.log(\`✅ 2. Anschreiben check: ${triggered} triggered, ${skipped} skipped, ${errors} errors out of ${filtered.length} eligible\`)`

**Export:** `module.exports = SecondLetterTriggerService;` (export the CLASS, not an instance — server.js will instantiate with dependencies).

Use `const { v4: uuidv4 } = require('uuid');` and `const Client = require('../models/Client');` at the top.
