Create `server/services/secondLetterCalculationService.js` with a single exported function `calculateSecondLetterFinancials(snapshot, creditors)`.

**Implementation — follow this exact structure:**

1. **Require and instantiate calculator:**
   ```javascript
   const GermanGarnishmentCalculator = require('./germanGarnishmentCalculator');
   const calculator = new GermanGarnishmentCalculator();
   ```

2. **Function signature:**
   ```javascript
   function calculateSecondLetterFinancials(snapshot, creditors)
   ```
   - `snapshot`: The `second_letter_financial_snapshot` object (from Phase 28/30)
   - `creditors`: The `client.final_creditor_list` array
   - Returns: `{ success, garnishableAmount, planType, totalDebt, creditorCalculations, error? }`

3. **Step 1 — CALC-01: Garnishable amount from snapshot:**
   - Call `calculator.calculate(snapshot.monthly_net_income, snapshot.familienstand, snapshot.anzahl_unterhaltsberechtigte || 0)`
   - Use `calculate()` method — NOT `calculateGarnishableAmount()` (does not exist, latent bug in adminFinancialController)
   - If `!garnishmentResult.success`, return `{ success: false, error: ... }`
   - Clamp result: `Math.max(0, garnishmentResult.garnishableAmount)` — cannot be negative

4. **Step 2 — CALC-02: Plan type:**
   - `const planType = garnishableAmount > 0 ? 'RATENPLAN' : 'NULLPLAN'`
   - MUST use uppercase enum values per v10 locked decisions (not lowercase 'ratenplan')

5. **Step 3 — CALC-03: Validate creditors + total debt:**
   - Check every creditor has non-null `claim_amount`. If any is null/undefined, return `{ success: false, error: 'Creditor "X" is missing claim_amount — calculation aborted' }`. This is a locked decision — missing claim_amount aborts.
   - Compute `totalDebt = creditors.reduce((sum, c) => sum + c.claim_amount, 0)`
   - If `totalDebt === 0`, return `{ success: false, error: 'Total debt is zero — cannot calculate creditor quotas' }`

6. **Step 4 — CALC-04: Per-creditor calculations:**
   - Map over creditors to produce `creditorCalculations` array
   - For NULLPLAN: `tilgungsangebot = 0` explicitly (locked decision — uniform data structure)
   - For RATENPLAN: `tilgungsangebot = (creditor.claim_amount / totalDebt) * garnishableAmount`
   - Round: `Math.round(tilgungsangebot * 100) / 100` — consistent with germanGarnishmentCalculator's own rounding (line 422)
   - Do NOT use `toFixed()` — returns a string, breaks MongoDB Number field
   - Quota percentage: `Math.round((creditor.claim_amount / totalDebt) * 10000) / 100` — e.g. 33.33%
   - Guard every result with `Number.isFinite()` — if any is not finite, return `{ success: false, error: 'Non-finite value...' }`
   - Each entry: `{ creditor_id, creditor_name, claim_amount, tilgungsangebot, quota_percentage }`
   - `creditor_id`: use `creditor._id?.toString()` (MongoDB ObjectId) — NOT `creditor.id`
   - `creditor_name`: use `creditor.sender_name || creditor.creditor_name || creditor.glaeubiger_name || 'Unbekannt'`

7. **Return shape on success:**
   ```javascript
   { success: true, garnishableAmount, planType, totalDebt, creditorCalculations }
   ```

8. **Module export:**
   ```javascript
   module.exports = { calculateSecondLetterFinancials };
   ```

**Critical rules:**
- Read EXCLUSIVELY from snapshot fields, never from live `financial_data`
- Use `_id` for MongoDB document references (not `id`) per CLAUDE.md
- Single creditor = normal calculation (100% quota, full garnishable amount as Tilgungsangebot)
- Euro float storage (consistent with existing `garnishable_amount` in `financial_data`)
- Add JSDoc with clear parameter and return type documentation
