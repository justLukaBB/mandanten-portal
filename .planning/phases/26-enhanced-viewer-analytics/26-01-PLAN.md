**Architecture:**
- Detect content type from the response `Content-Type` header (or file extension fallback)
- Route to: `PdfViewer` (for `application/pdf`), `ImageViewer` (for `image/*`), or `IframeFallback` (for everything else)
- Wrap everything in a container with the existing border/radius styling

**PDF Viewer (`PdfViewer` internal component):**
- Set `pdfjsLib.GlobalWorkerOptions.workerSrc` to the pdfjs-dist worker bundled via `new URL('pdfjs-dist/build/pdf.worker.mjs', import.meta.url).href` (Vite-compatible pattern)
- Load PDF from ArrayBuffer using `pdfjsLib.getDocument({ data: arrayBuffer })`
- Render ALL pages stacked vertically in a scrollable container (continuous scroll). Each page gets its own canvas.
- Default zoom: fit-to-width — calculate scale as `containerWidth / pageWidth`
- Zoom state: `scale` (number), recalculate on zoom change
- Re-render all canvases when scale changes using `page.render({ canvasContext, viewport: page.getViewport({ scale }) })`
- Loading: show centered spinner (use a simple CSS spinner) with percentage (pages loaded / total pages * 100)
- Error handling: if `pdfjsLib.getDocument()` or `page.render()` throws, set `pdfError` state and render `IframeFallback` instead (silent fallback per user decision)
- Cleanup: cancel any pending render tasks on unmount or document change

**Image Viewer (`ImageViewer` internal component):**
- Render the image in a scrollable container with CSS transform for zoom (transform: scale(N))
- Pan: track mouse drag with `onMouseDown/onMouseMove/onMouseUp` for panning (translateX/translateY)
- Default: fit-to-width (object-fit: contain, max-width: 100%)
- Share the same zoom level state as the toolbar controls

**Toolbar (fixed top row, always visible):**
- Use lucide-react icons: `ZoomIn`, `ZoomOut`, `Download`, `Printer` (or `FileDown`, `Printer`)
- Layout: flex row with border-bottom, padding 8px 16px, gap between groups
- Left group: Zoom out button (`-`), percentage Select dropdown (50%, 75%, 100%, 150%, 200%, and "Anpassen" for fit-to-width), Zoom in button (`+`)
- Use the shadcn/ui `Select` component for the percentage dropdown. Import from `./ui/select`.
- Right group: Download button, Print button
- Zoom in/out buttons step through the preset values: [0.5, 0.75, 1.0, 1.5, 2.0]
- "Anpassen" (fit-to-width) is the default and recalculates based on container width
- Download: create an `<a>` tag with the blob URL and `download` attribute, trigger click
- Print: open blob URL in a new window and call `window.print()` on it
- Styling: bg white, border-bottom 1px #E5E7EB, icons size 16, buttons 28x28 with hover bg #F3F4F6, rounded-md

**States (same as current but improved):**
- No document: file icon + "Kein Dokument ausgewahlt" (keep existing design)
- Loading: centered spinner with "Dokument wird geladen... X%" text
- Error (non-PDF): red error box (keep existing design)
- PDF.js failure: silently render IframeFallback instead

**Key constraints:**

- Store the raw `ArrayBuffer` (for PDF) or `Blob URL` (for images/fallback) based on content type detection
- Always reset zoom to fit-to-width when `documentName` changes (per user decision)
