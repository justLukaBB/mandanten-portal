**IMPORTANT:** Phase 25 creates `server/controllers/adminReviewController.js` and `server/routes/admin-review.js`. If Phase 25 has not been executed yet, these files may not exist. In that case, create them with the analytics endpoint AND the basic structure that Phase 25 would have created (admin auth middleware, router factory pattern matching `createAgentReviewRouter` in agent-review.js). Check `server/server.js` to see if admin-review route is already registered — if not, register it.

Add a `getAnalytics` handler to `adminReviewController.js`:

1. Accept `dateRange` query param: `7`, `30`, `90`, or `all` (default: `30`)
2. Compute date filter: `new Date(Date.now() - days * 86400000)` for numeric values; no filter for `all`
3. Query MongoDB Client collection for reviewed clients (those with `final_creditor_list` entries that have `review_action` set):

**KPI calculations:**
- `totalReviews`: Count of distinct clients where at least one creditor has `review_action` set, within date range (use `reviewed_at` field)
- `pending`: Count of clients matching the available-clients query (needs_review conditions — reuse the same filter from `getAvailableClients` in agentReviewController.js)
- `avgProcessingTime`: Average difference between `reviewed_at` and case creation `created_at` in hours, for clients reviewed within the date range. Return as formatted string like "2.5h"
- `autoApprovedPercent`: Percentage of reviewed creditors where `review_action === 'confirm'` (i.e., AI was correct, no correction needed). Round to 1 decimal.

**Chart data:**
- `reviewsPerDay`: Array of `{ date: 'YYYY-MM-DD', count: number }` — aggregate reviews by day within date range. Use MongoDB `$group` with `$dateToString`.
- `confidenceDistribution`: Array of `{ range: '0-50%' | '50-80%' | '80-100%', count: number }` — bucket creditor confidence values.
- `outcomeBreakdown`: Array of `{ name: 'Bestätigt' | 'Korrigiert' | 'Übersprungen', value: number }` — count review_actions (confirm → Bestätigt, correct → Korrigiert, skip → Übersprungen). German labels.
- `agentPerformance`: Array of `{ agent: string, reviewsHandled: number, avgTime: string, accuracy: number }` — group by `reviewed_by` field. `accuracy` = confirm count / total for that agent * 100.

Return JSON: `{ success: true, data: { kpi: { totalReviews, pending, avgProcessingTime, autoApprovedPercent }, charts: { reviewsPerDay, confidenceDistribution, outcomeBreakdown, agentPerformance } } }`

4. Add route in `admin-review.js`: `router.get('/analytics', authenticateAdmin, controller.getAnalytics)`

