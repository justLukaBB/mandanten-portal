1. **src/store/slices/authSlice.ts** — Create the auth state slice:
   - State shape: `{ token: string | null, loginTimestamp: number | null, isAuthenticated: boolean }`
   - On app load, initialize state by reading from localStorage:
     - `token = localStorage.getItem('admin_token')`
     - `loginTimestamp = Number(localStorage.getItem('login_timestamp')) || null`
     - `isAuthenticated`: true if token exists AND loginTimestamp is within 8 hours of Date.now() (8 * 60 * 60 * 1000 = 28800000 ms)
   - Actions:
     - `login({ token, timestamp })`: Sets token, loginTimestamp, isAuthenticated=true. Writes `admin_token` and `login_timestamp` to localStorage.
     - `logout()`: Clears token, loginTimestamp, isAuthenticated=false. Removes `admin_token` and `login_timestamp` from localStorage.
   - Selectors: `selectIsAuthenticated`, `selectToken`, `selectLoginTimestamp`
   - Export the 8-hour constant: `export const SESSION_DURATION_MS = 8 * 60 * 60 * 1000;`
   - Helper function `isSessionValid(timestamp: number | null): boolean` that checks if timestamp is within SESSION_DURATION_MS of Date.now(). Export this for reuse in Plan 20-02.

2. **src/store/api/authApi.ts** — Create login mutation using baseApi:
   - Import `baseApi` from `./baseApi`
   - Use `baseApi.injectEndpoints` to add a `login` mutation:
     - `query: (credentials: { email: string; password: string }) => ({ url: '/api/admin/login', method: 'POST', body: credentials })`
   - The backend POST /api/admin/login returns the token. Check the response shape — it likely returns `{ token: string }` or `{ admin_token: string }`. The mutation just passes through the response; the LoginPage component handles extracting the token and dispatching the login action.
   - Export `useLoginMutation`

3. **src/store/index.ts** — Add authSlice to the store:
   - Import `authSlice` from `./slices/authSlice`
   - Add `auth: authSlice.reducer` to the reducer map alongside the existing `[baseApi.reducerPath]: baseApi.reducer`
