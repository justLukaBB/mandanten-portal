Protect all app routes behind authentication, add the /login route, wire up the logout button in the sidebar, and add a 401 response interceptor that auto-logs out expired sessions.

Purpose: Without route protection, anyone can access /clients by typing the URL. Without a logout button, admins can never sign out. Without a 401 interceptor, expired tokens result in silent API failures instead of a clear redirect to login.

Output: A ProtectedRoute wrapper used on all content routes, a /login route for the LoginPage, a logout button in the sidebar, and a baseQuery wrapper that catches 401 responses.

2. **src/store/api/baseApi.ts** — Add a reauth wrapper around the baseQuery:
   - Create a `baseQueryWithReauth` function that wraps the existing `fetchBaseQuery`:
     ```typescript
     import type { BaseQueryFn, FetchArgs, FetchBaseQueryError } from '@reduxjs/toolkit/query/react';
     import { logout, isSessionValid, selectLoginTimestamp } from '../slices/authSlice';

     const rawBaseQuery = fetchBaseQuery({
       baseUrl: import.meta.env.VITE_API_BASE_URL || '',
       prepareHeaders: (headers, { getState }) => {
         const token = (getState() as RootState).auth.token;
         if (token) {
           headers.set('Authorization', `Bearer ${token}`);
         }
         return headers;
       },
     });

     const baseQueryWithReauth: BaseQueryFn<string | FetchArgs, unknown, FetchBaseQueryError> = async (args, api, extraOptions) => {
       // Check session validity before making request
       const state = api.getState() as RootState;
       const loginTimestamp = state.auth.loginTimestamp;
       if (!isSessionValid(loginTimestamp)) {
         api.dispatch(logout());
         // Return an error so the calling component knows auth failed
         return { error: { status: 401, data: 'Session expired' } as FetchBaseQueryError };
       }

       const result = await rawBaseQuery(args, api, extraOptions);

       // If backend returns 401, the token is invalid — log out
       if (result.error && result.error.status === 401) {
         api.dispatch(logout());
       }

       return result;
     };
     ```
   - Update `createApi` to use `baseQueryWithReauth` instead of the raw `fetchBaseQuery`
   - **IMPORTANT:** Also update `prepareHeaders` to read the token from Redux state (`getState().auth.token`) instead of directly from localStorage. The authSlice is the single source of truth — this keeps the store and the header in sync.
   - Import `RootState` type from the store. Since this creates a circular import (store imports baseApi, baseApi imports from store), use a type-only import: `import type { RootState } from '../index'` or inline the type. Alternatively, keep reading from localStorage in prepareHeaders (avoids circular import) and only use Redux state for the session validity check. Either approach is acceptable — choose the one that avoids circular import issues.
