**New query — getAdminReviewQueue:**
```ts
getAdminReviewQueue: builder.query<AdminReviewQueueResponse, ReviewQueueParams>({
  query: ({ page = 1, limit = 25, search = '', priority = 'all' } = {}) => {
    const params = new URLSearchParams({
      page: page.toString(),
      limit: limit.toString(),
    });
    if (search) params.append('search', search);
    if (priority && priority !== 'all') params.append('priority', priority);
    return { url: `/api/admin/review/queue?${params.toString()}`, method: 'GET' };
  },
  providesTags: ['ReviewQueue'],
}),
```

**New mutations:**

```ts
assignReview: builder.mutation<{ success: boolean; client_id: string; assigned_to: string }, { clientId: string; assigned_to: string }>({
  query: ({ clientId, assigned_to }) => ({
    url: `/api/admin/review/${clientId}/assign`,
    method: 'POST',
    body: { assigned_to },
  }),
  invalidatesTags: ['ReviewQueue'],
}),

unassignReview: builder.mutation<{ success: boolean; client_id: string }, string>({
  query: (clientId) => ({
    url: `/api/admin/review/${clientId}/assign`,
    method: 'DELETE',
  }),
  invalidatesTags: ['ReviewQueue'],
}),

batchAssign: builder.mutation<{ success: boolean; updated_count: number }, { client_ids: string[]; assigned_to: string }>({
  query: (body) => ({
    url: `/api/admin/review/batch/assign`,
    method: 'POST',
    body,
  }),
  invalidatesTags: ['ReviewQueue'],
}),

batchUpdatePriority: builder.mutation<{ success: boolean; updated_count: number }, { client_ids: string[]; priority: string }>({
  query: (body) => ({
    url: `/api/admin/review/batch/priority`,
    method: 'POST',
    body,
  }),
  invalidatesTags: ['ReviewQueue'],
}),

batchConfirm: builder.mutation<{ success: boolean; confirmed_count: number }, { client_ids: string[] }>({
  query: (body) => ({
    url: `/api/admin/review/batch/confirm`,
    method: 'POST',
    body,
  }),
  invalidatesTags: ['ReviewQueue'],
}),
```

**1. Create BatchActionBar.tsx**

Create a new component at `MandantenPortalDesign/src/app/components/review/BatchActionBar.tsx`.

**Design requirements (from Guidelines.md):**
- Sticky bottom bar: `position: fixed; bottom: 0; left: 220px; right: 0` (220px = sidebar width)
- Background: `var(--surface)` (#FFFFFF) — card-like element
- Border-top: `1px solid var(--border)` (#E5E7EB)
- Padding: 12px 24px
- z-index: 50
- Flex row: selection count label on the left, action buttons on the right
- Use `framer-motion` AnimatePresence for slide-up entrance (translateY from 100% to 0, duration 150ms)

**Props:**
```ts
interface BatchActionBarProps {
  selectedIds: string[];
  onClearSelection: () => void;
}
```

**Content:**
- Left side: text "{N} ausgewählt" with a "Auswahl aufheben" ghost button (calls onClearSelection)
- Right side: 3 action buttons
  1. "Zuweisen" (secondary/outlined button) — opens a small dropdown/dialog to enter agent name, then calls `useBatchAssignMutation`
  2. "Priorität ändern" (secondary/outlined button) — opens dropdown with 3 options (Hoch/Mittel/Niedrig), calls `useBatchUpdatePriorityMutation`
  3. "Bestätigen" (primary/orange CTA button — max 1 per section) — calls `useBatchConfirmMutation` with confirmation dialog first

**Interaction flow for each batch action:**
- Zuweisen: clicking shows a `DropdownMenu` (shadcn/ui) with an input field for agent name + "Zuweisen" button. On submit, calls batchAssign({ client_ids: selectedIds, assigned_to }). Show sonner toast on success "N Fälle zugewiesen". Clear selection.
- Priorität ändern: clicking shows DropdownMenu with 3 items (Hoch, Mittel, Niedrig). On click, calls batchUpdatePriority({ client_ids: selectedIds, priority }). Toast "Priorität für N Fälle aktualisiert". Clear selection.
- Bestätigen: clicking shows a confirm dialog (use window.confirm or a simple Dialog) "N Fälle bestätigen? Alle Gläubiger in diesen Fällen werden als geprüft markiert." On confirm, calls batchConfirm({ client_ids: selectedIds }). Toast "N Fälle bestätigt". Clear selection.

After each batch operation succeeds, the RTK Query cache invalidation via 'ReviewQueue' tag will auto-refetch the queue.

**Error handling:** If any mutation fails, show sonner error toast with the error message. Do NOT clear selection on error (let user retry).
