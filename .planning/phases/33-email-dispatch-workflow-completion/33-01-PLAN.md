<objective>
Build the second letter email dispatch service and admin endpoint. The service iterates over each creditor in final_creditor_list, sends the pre-generated DOCX via creditorEmailService.sendSecondRoundEmail(), updates per-creditor MongoDB tracking fields, appends Zendesk audit comments, and atomically transitions second_letter_status from FORM_SUBMITTED to SENT when all sends succeed. Retry logic (3 attempts per creditor) and admin alerting on exhaustion are included.

Purpose: This is the dispatch layer of the 2. Anschreiben workflow — the final step before the admin UI. Without this, generated DOCX documents sit on disk but are never sent to creditors.

Output: secondLetterService.js (service), admin-second-letter.js (route), server.js (registration)

**Constructor:**
- Accept dependencies: `{ Client, creditorEmailService, ZendeskManager }` (dependency injection, consistent with creditorContactService pattern)

**dispatchSecondLetterEmails(clientId) method:**

1. **Load client with status guard:**
   ```javascript
   const client = await Client.findById(clientId);
   if (!client) throw new Error('Client not found');
   if (client.second_letter_status !== 'FORM_SUBMITTED') {
     return { success: false, error: 'INVALID_STATUS', message: `Client status is ${client.second_letter_status}, expected FORM_SUBMITTED` };
   }
   ```

2. **Filter creditors to dispatch:**
   - Filter `client.final_creditor_list` for creditors that have:
     - A valid `sender_email` (truthy, non-empty string)
     - A `second_letter_document_filename` (set by Phase 32)
   - Track skipped creditors (no email or no document) separately — these are expected skips, not failures
   - Log count: `{N} creditors to dispatch, {M} skipped (no email or no document)`

3. **Per-creditor dispatch loop with retry:**
   - For each eligible creditor, attempt up to 3 times:
     - Build full document path: `path.join(__dirname, '../generated_documents/second_round/', creditor.second_letter_document_filename)`
     - Verify file exists with `fs.existsSync()` before calling send — if missing, treat as failure (do NOT retry, file won't appear)
     - Call `creditorEmailService.sendSecondRoundEmail()` with:
       - `recipientEmail`: `creditor.sender_email`
       - `recipientName`: `creditor.creditor_name || creditor.sender_name`
       - `clientName`: `${client.firstName} ${client.lastName}`
       - `clientReference`: `client.aktenzeichen`
       - `attachment`: `{ filename: creditor.second_letter_document_filename, path: fullDocPath }`
     - On success (`result.success === true`):
       - Update per-creditor tracking in MongoDB (SEND-02):
         ```javascript
         await Client.updateOne(
           { _id: client._id, 'final_creditor_list._id': creditor._id },
           { $set: {
             'final_creditor_list.$.second_letter_sent_at': new Date(),
             'final_creditor_list.$.second_letter_email_sent_at': new Date(),
             'final_creditor_list.$.second_letter_document_filename': creditor.second_letter_document_filename
           }}
         );
         ```
       - Add Zendesk audit comment (SEND-03) — wrapped in try/catch, non-blocking:
         - Resolve ticket ID: `client.zendesk_ticket_id || creditor.main_zendesk_ticket_id`
         - If ticket ID exists, call `zendesk.addTicketComment(ticketId, commentBody, false)` with German comment body including: recipient name, email, document filename, Resend email ID, timestamp
         - If no ticket ID or Zendesk fails, console.warn only — do NOT fail the send
       - Break retry loop
     - On failure: store error, continue to next retry attempt
   - After 3 failed attempts: call `triggerAdminAlert(client, creditor, lastError)` (SEND-05), set `allSucceeded = false`
   - 2-second delay between creditors (not between retries): `await new Promise(resolve => setTimeout(resolve, 2000))`

4. **Atomic SENT transition (SEND-04):**
   - Only if `allSucceeded === true`:
     ```javascript
     const updated = await Client.findOneAndUpdate(
       { _id: client._id, second_letter_status: 'FORM_SUBMITTED' },
       { $set: { second_letter_status: 'SENT', second_letter_sent_at: new Date() } },
       { new: true }
     );
     ```
   - If `!updated`: log warning (status guard — another process already transitioned)
   - If `allSucceeded === false`: status stays FORM_SUBMITTED (per SEND-05)

5. **Return result object:**
   ```javascript
   return {
     success: allSucceeded,
     totalCreditors: client.final_creditor_list.length,
     dispatched: successCount,
     failed: failedCount,
     skipped: skippedCount,
     status: allSucceeded ? 'SENT' : 'FORM_SUBMITTED'
   };
   ```

- Add Zendesk internal comment to main ticket (non-blocking try/catch): warning comment with creditor name, error message, "manuelle Nachbearbeitung erforderlich"

**Part A: Create `server/routes/admin-second-letter.js`**

Use the established route factory pattern (consistent with `admin-settlement.js`, `admin-dashboard.js`):

```javascript
const express = require('express');
const { authenticateAdmin } = require('../middleware/auth');
const SecondLetterService = require('../services/secondLetterService');
const ZendeskManager = require('../services/zendeskManager');

module.exports = ({ Client, creditorEmailService }) => {
  const router = express.Router();
  const secondLetterService = new SecondLetterService({
    Client,
    creditorEmailService,
    ZendeskManager
  });

  // POST /api/admin/clients/:clientId/send-second-letter
  router.post('/clients/:clientId/send-second-letter',
    authenticateAdmin,
    async (req, res) => {
      try {
        const { clientId } = req.params;
        const result = await secondLetterService.dispatchSecondLetterEmails(clientId);

        if (!result.success && result.error === 'INVALID_STATUS') {
          return res.status(409).json({ success: false, error: result.message });
        }

        const statusCode = result.success ? 200 : 207; // 207 Multi-Status for partial failure
        return res.status(statusCode).json({
          success: result.success,
          dispatched: result.dispatched,
          failed: result.failed,
          skipped: result.skipped,
          totalCreditors: result.totalCreditors,
          status: result.status
        });
      } catch (error) {
        console.error('Error dispatching second letter emails:', error);
        return res.status(500).json({ success: false, error: error.message });
      }
    }
  );

  return router;
};
```

Key decisions:
- 409 Conflict for invalid status (client not in FORM_SUBMITTED) — consistent with Phase 6's HTTP 409 guard pattern
- 207 Multi-Status when some creditors failed but some succeeded — the admin knows partial send occurred
- 200 when all succeeded
- authenticateAdmin middleware for access control
- Use `_id` (not `id`) for MongoDB lookups per CLAUDE.md
