  <action>
Create a new React component `EditableCell` that handles inline cell editing with auto-save on blur.

**Props interface:**
```typescript
interface EditableCellProps {
  value: string;                    // Current display value
  fieldName: string;                // German field name to send to PUT endpoint (e.g., 'glaeubiger_name')
  clientId: string;                 // For the PUT URL
  creditorId: string;               // For the PUT URL
  onSaved?: (newValue: string) => void;  // Callback to update parent state after save
  type?: 'text' | 'boolean' | 'textarea';  // Input type (default: text)
  fallbackValue?: string;           // Fallback display when value is empty/N/A (default: 'N/A')
}
```

**Behavior:**
1. **Display mode (default):** Render the value as a plain `<span>` with a subtle hover effect (light background, cursor pointer) to signal clickability. If value equals fallbackValue or is empty, show it in muted gray.
2. **Edit mode (on click):** Replace the span with an `<input>` (or `<select>` for boolean type). Auto-focus the input. For boolean type, render a `<select>` with options "Ja"/"Nein".
3. **Save on blur:** When the input loses focus, compare the new value to the original. If unchanged, silently return to display mode. If changed, send a PUT request to `/api/admin/clients/${clientId}/creditors/${creditorId}` with body `{ [fieldName]: newValue }`. Use the `api` instance from `../../config/api`.
4. **Save on Enter key:** Also trigger save when Enter is pressed.
5. **Cancel on Escape:** Revert to original value and return to display mode.
6. **Success feedback:** After successful PUT response, show a green checkmark icon (CheckCircleIcon from heroicons) next to the value for 1.5 seconds, then fade it out. Update the displayed value to the new value.
7. **Error feedback:** After failed PUT response, keep the cell in edit mode with a red border, show a small red error text below the input ("Speichern fehlgeschlagen"). The unsaved value is retained so the admin can retry or press Escape to cancel.
8. **Loading state:** While the PUT request is in flight, show a subtle spinner (CSS-only, no extra dependency) and disable the input.

**Styling notes:**
- Match the existing table text size (text-xs, text-gray-800 from UserDetailView)
- Input styling: `border border-gray-300 rounded px-2 py-1 text-xs focus:ring-1 focus:ring-red-700 focus:border-red-700`
- Error input: add `border-red-500` class
- Success checkmark: `text-green-500 w-3 h-3 inline ml-1` with CSS transition (opacity 0 to 1 then back to 0)
- Hover effect on display mode: `hover:bg-gray-50 cursor-pointer rounded px-1 -mx-1`

Import `api` from `../../config/api` and `CheckCircleIcon` from `@heroicons/react/24/outline`.

**Add a handler to refresh creditor data after save.** Create a callback function that updates the user state's `final_creditor_list` in place when a cell saves successfully:
```typescript
const handleCreditorFieldSaved = (creditorId: string, fieldName: string, newValue: string) => {
  setUser(prev => {
    if (!prev) return prev;
    return {
      ...prev,
      final_creditor_list: prev.final_creditor_list?.map(c =>
        c.id === creditorId ? { ...c, [fieldName]: newValue } : c
      )
    };
  });
};
```

**Replace each `<td>` in the creditor table body.** For each editable column, replace the static `<td>` with an EditableCell. The mapping of columns to fieldNames and values:

For the `boolean` type (needs_manual_review), EditableCell should send `true`/`false` boolean to the backend, not the string "Ja"/"Nein". Handle this mapping inside EditableCell for `type="boolean"`.

For `review_reasons`, since the backend expects an Array, the EditableCell `onSaved` callback for this field should split the comma-separated string and send `{ review_reasons: value.split(',').map(s => s.trim()).filter(Boolean) }`. Handle this as a special case: add an optional `transformBeforeSend` prop to EditableCell that transforms the string value before sending. For review_reasons, pass: `transformBeforeSend={(v) => v.split(',').map(s => s.trim()).filter(Boolean)}`.

The 2 read-only columns (Anzahl Dokumente and Quell-Dokumente) remain as plain `<td>` elements unchanged.

**Important:** The `userId` prop is the client identifier used throughout UserDetailView for API calls. Pass it as `clientId` to EditableCell. The `c.id` is the creditor ID within `final_creditor_list`.
