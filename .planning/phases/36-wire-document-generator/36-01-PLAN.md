Modify the `send-second-letter` POST handler (lines 98-127) in `server/routes/admin-second-letter.js` to generate DOCX documents before dispatching emails. This follows the established first-round pattern from `creditorContactService.js` (lines 154-179) where generation precedes dispatch.

Steps:

1. Add require at the top of the file (after existing requires, around line 9):
   ```javascript
   const SecondLetterDocumentGenerator = require('../services/secondLetterDocumentGenerator');
   ```

2. Replace the send-second-letter handler body (lines 100-126) with this sequence:

   a. Load client from DB:
   ```javascript
   const client = await Client.findById(clientId);
   if (!client) return res.status(404).json({ success: false, error: 'Client not found' });
   ```

   b. Status guard BEFORE generation (fail fast — don't generate for wrong status):
   ```javascript
   if (client.second_letter_status !== 'FORM_SUBMITTED') {
     return res.status(409).json({ success: false, error: 'INVALID_STATUS', message: `Status is ${client.second_letter_status}, expected FORM_SUBMITTED` });
   }
   ```

   c. Snapshot guard (check calculation_status is completed):
   ```javascript
   const snapshot = client.second_letter_financial_snapshot;
   if (!snapshot || snapshot.calculation_status !== 'completed') {
     return res.status(400).json({ success: false, error: 'Snapshot not ready — run recalculate-second-letter first' });
   }
   ```

   d. Generate DOCX for all creditors:
   ```javascript
   const generator = new SecondLetterDocumentGenerator();
   const genResult = await generator.generateForAllCreditors(client, snapshot);
   console.log(`[SecondLetter] Generation: ${genResult.total_generated} generated, ${genResult.total_failed} failed`);
   if (genResult.total_generated === 0 && genResult.total_failed > 0) {
     return res.status(500).json({ success: false, error: 'Document generation failed for all creditors', details: genResult.errors });
   }
   ```

   e. Dispatch emails (existing call — unchanged):
   ```javascript
   const result = await secondLetterService.dispatchSecondLetterEmails(clientId);
   ```

   f. Keep the existing response handling (409/422/207/200) exactly as is.

Important: The `dispatchSecondLetterEmails()` call re-loads the client from DB internally (line 48 of secondLetterService.js), so it will pick up the `second_letter_document_filename` values that `generateForAllCreditors` just persisted to MongoDB. No parameter changes needed.

The dispatch service's own internal status guard (line 53) may seem redundant now, but keep it — defense in depth. Both guards use the same check (`second_letter_status !== 'FORM_SUBMITTED'`).

  <files>server/services/secondLetterService.js</files>
  <action>
Fix the path mismatch bug where `secondLetterService.js` looks for files at `second_round/{filename}` but `SecondLetterDocumentGenerator` saves them at `second_round/{clientId}/{filename}`.

The fix is in `dispatchSecondLetterEmails()` around line 101-105.

Current code (line 101 + 105):
```javascript
const GENERATED_DOCS_DIR = path.join(__dirname, '../generated_documents/second_round');
// ...
const fullDocPath = path.join(GENERATED_DOCS_DIR, creditor.second_letter_document_filename);
```

Fix: Include the clientId subdirectory in the path construction. The `clientId` parameter is already available (line 42 — function parameter). Use `client._id.toString()` since client is loaded at line 48.

Replace line 101:
```javascript
const GENERATED_DOCS_DIR = path.join(__dirname, '../generated_documents/second_round');
```
with:
```javascript
const clientIdStr = client._id.toString();
const GENERATED_DOCS_DIR = path.join(__dirname, '../generated_documents/second_round', clientIdStr);
```

This way `fullDocPath` at line 105 becomes `second_round/{clientId}/{filename}` which matches the generator's output path pattern (confirmed in secondLetterDocumentGenerator.js line 320: `path.join(__dirname, '../generated_documents/second_round', clientId)`).

Important: Do NOT change line 105 (`const fullDocPath = path.join(GENERATED_DOCS_DIR, creditor.second_letter_document_filename);`) — it stays the same. The fix is only in how `GENERATED_DOCS_DIR` is constructed.

The `second_letter_document_filename` field on creditors stores ONLY the bare filename (e.g. `Mueller_AZ-12345_Ratenplan.docx`), NOT a path with clientId. This is correct and consistent with what the field name implies.
