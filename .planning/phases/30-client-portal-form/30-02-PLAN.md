**Create src/pages/SecondLetterForm.tsx** — a standalone page component. Follow the FinancialDataForm.tsx pattern for state management, validation, and submit flow. Follow the PersonalPortal.tsx inline modal pattern for the confirmation dialog.

**CRITICAL: This is in /src/ (CRA portal), NOT in MandantenPortalDesign/src/.** Per user decision: "Portal form: in /src/ (CRA)".

**Token handling:**
- Read token from URL query string: `useSearchParams` from 'react-router-dom', `searchParams.get('token')`
- Do NOT store token in localStorage — pass it explicitly in the Authorization header for every API call
- Use axios directly (import from '../config/api' or plain axios) with explicit `{ headers: { Authorization: \`Bearer ${token}\` } }` — this prevents the api.ts interceptor from attaching a wrong stored token

**Page states (useState):**
- `status`: 'loading' | 'form' | 'already_submitted' | 'unavailable' | 'success' | 'error'
- `preFillData`: pre-fill response from GET endpoint
- `formData`: current form field values
- `errors`: field-level validation errors (object)
- `isSubmitting`: boolean
- `showConfirmDialog`: boolean
- `submittedAt`: date string for already-submitted display

**On mount (useEffect with token dependency):**
1. If no token in URL → set status 'error'
2. Call GET /api/second-letter-form with Bearer token header
3. If response.data.second_letter_status === 'PENDING' → set status 'form', populate formData from pre_fill
4. If response.data.second_letter_status === 'FORM_SUBMITTED' or 'SENT' → set status 'already_submitted', store submittedAt from second_letter_form_submitted_at
5. Any other status (IDLE) → set status 'unavailable'
6. On 401/network error → set status 'error' or 'unavailable'

**Form layout (per locked decisions):**
- Mobile-first design: max-w-lg mx-auto, responsive padding
- Top: Kanzlei branding — show the law firm name (can be hardcoded "Schuldenanalyse24" or read from config if available) + brief explanation text: "Bitte überprüfen und bestätigen Sie Ihre aktuellen Finanzdaten für das 2. Gläubigeranschreiben."
- Logical sections with section headings (h2 or h3):

  **Section 1: "Einkommensdaten"**
  - Monatliches Nettoeinkommen (number input, required, pre-filled from monthly_net_income)
  - Einkommensquelle (select: Angestellt, Selbstständig, Arbeitslos, Rentner, In Ausbildung — map to values: angestellt, selbststaendig, arbeitslos, rentner, in_ausbildung — pre-filled from income_source)

  **Section 2: "Persönliche Daten"**
  - Familienstand (select: Ledig, Verheiratet, Geschieden, Verwitwet — values: ledig, verheiratet, geschieden, verwitwet — pre-filled from marital_status)
  - Anzahl Unterhaltspflichten (number input, integer >= 0, pre-filled from number_of_dependents)

  **Section 3: "Weitere Angaben"**
  - Aktuelle Lohnpfändungen (toggle/checkbox: Ja/Nein — pre-filled from active_garnishments)
  - Neue Gläubiger hinzugekommen? (toggle/checkbox: Ja/Nein — always pre-filled as false)
    - When "Ja": animated slide-in container (`transition-all duration-200 overflow-hidden`, use max-h-0 / max-h-[600px]) showing:
      - Dynamic list of new creditor entries, each with:
        - Gläubiger Name (text input, required when visible)
        - Forderungsbetrag (number input, required when visible, in EUR)
        - Remove button (X) for entries beyond the first
      - "Weiteren Gläubiger hinzufügen" button to add more entries (starts with one entry)

  **Section 4: "Bestätigung"**
  - Checkbox: "Ich bestätige, dass die obigen Angaben korrekt und vollständig sind." (required — must be checked to submit)

- Submit button: "Daten einreichen" — full width on mobile, normal on desktop. Disabled when isSubmitting or confirmation checkbox unchecked.

**Validation (per locked decision — inline on blur + full on submit):**
- `validateField(fieldName)` — called onBlur for each field. Validates single field, sets error for that field.
- `validateForm()` — called on submit. Validates all fields. Returns errors object.
- Required fields: monthly_net_income (must be > 0), income_source (must be one of enum values), marital_status (must be non-empty), number_of_dependents (must be >= 0 integer), active_garnishments (boolean, always valid), has_new_creditors (boolean, always valid), confirmation (must be true).
- If has_new_creditors === true: each new_creditors entry must have non-empty name and amount > 0.
- Error display: red text below each field with the error message. Use red border on the field.

**Submit flow (per locked decisions):**
1. User clicks "Daten einreichen" → run validateForm()
2. If validation passes → show confirmation dialog (showConfirmDialog = true)
3. Confirmation dialog is an inline modal (fixed overlay, bg-black bg-opacity-50, centered white card, z-50, p-4 for mobile):
   - Title: "Daten einreichen?"
   - Text: "Ihre Daten können nach dem Absenden nicht mehr geändert werden. Möchten Sie fortfahren?"
   - Two buttons: "Bestätigen" (primary/orange) and "Abbrechen" (secondary/gray)
4. User clicks "Bestätigen" → set isSubmitting true, hide dialog, form becomes non-editable (pointer-events-none + opacity-50), submit button shows spinner
5. POST /api/second-letter-form with Bearer token header + form data body
6. On success (response.data.success) → set status 'success'
7. On 409 (already submitted) → set status 'already_submitted'
8. On error → set isSubmitting false, show red error banner at top of form: "Es ist ein Fehler aufgetreten. Bitte versuchen Sie es erneut." — form data is preserved, user can retry

**Success state (per locked decision):**
- Form disappears entirely
- Show: green checkmark icon (CheckCircleIcon from @heroicons/react/24/solid or an SVG), heading "Vielen Dank!", text "Ihre Finanzdaten wurden erfolgreich eingereicht. Sie können dieses Fenster nun schließen."
- Same Kanzlei branding as form page (consistent appearance)

**Already-submitted state (per locked decision):**
- No form, no data display
- Show: info icon, heading "Bereits eingereicht", text "Ihre Daten wurden am [formatted date] eingereicht." — format date with `new Date(submittedAt).toLocaleDateString('de-DE')`
- Same Kanzlei branding

**Unavailable state (per locked decision):**
- Friendly message: "Dieses Formular ist derzeit nicht verfügbar." — no technical details
- Same Kanzlei branding

**Error state (no token / network error):**
- "Ein Fehler ist aufgetreten. Bitte verwenden Sie den Link aus Ihrer E-Mail."

**Styling notes:**
- This is the CRA portal with Tailwind v3 — use Tailwind utility classes
- Background: use the portal's existing bg pattern (likely bg-gray-50 or bg-white)
- Form card: bg-white rounded-lg shadow-sm p-6 on mobile, p-8 on desktop
- Section headings: text-lg font-semibold text-gray-900 mb-4
- Labels: text-sm font-medium text-gray-700
- Inputs: standard Tailwind form styling (border border-gray-300 rounded-md shadow-sm focus:ring focus:ring-blue-500)
- Error state: border-red-500 + text-red-600 text below
