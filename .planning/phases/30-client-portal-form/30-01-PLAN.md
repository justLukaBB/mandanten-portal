**In clientPortalController.js**, add two new handler methods to the exported controller object:

**handleGetSecondLetterFormData:**
1. Use `req.clientId` (set by authenticateSecondLetterToken middleware) to find the client: `const client = await Client.findOne({ id: req.clientId })` — return 404 if not found
2. Return JSON with:
   - `second_letter_status`: the client's current status
   - `second_letter_form_submitted_at`: timestamp if already submitted (for the "bereits eingereicht" display)
   - `pre_fill` object mapping each form field to its correct DB source (critical — use the exact mapping from research):
     - `monthly_net_income`: `client.financial_data?.monthly_net_income ?? null`
     - `marital_status`: `client.financial_data?.marital_status ?? client.familienstand ?? null`
     - `income_source`: `client.extended_financial_data?.berufsstatus ?? null`
     - `number_of_dependents`: `client.extended_financial_data?.anzahl_unterhaltsberechtigte ?? null`
     - `active_garnishments`: `client.aktuelle_pfaendung ?? false`
     - `new_creditors`: `false` (always false pre-fill — this is new data the form collects)
3. Use optional chaining throughout — financial_data or extended_financial_data may be null/undefined for clients who never filled earlier forms

**handleSubmitSecondLetterForm:**
1. Use `req.clientId` to find the client — return 404 if not found
2. **STATUS GUARD FIRST** (before any data writes): Check `client.second_letter_status !== 'PENDING'` → return 409 `{ error: 'Formular nicht verfügbar', code: 'NOT_PENDING' }`
3. **Validate required fields** from req.body: `monthly_net_income` (number), `income_source` (string, one of: 'angestellt', 'selbststaendig', 'arbeitslos', 'rentner', 'in_ausbildung'), `marital_status` (string), `number_of_dependents` (integer >= 0), `active_garnishments` (boolean), `has_new_creditors` (boolean), `confirmation` (must be true). If `has_new_creditors` is true, validate `new_creditors` array has at least one entry with `name` (non-empty string) and `amount` (number > 0). Return 400 with specific field errors if validation fails.
4. Use `safeClientUpdate(client.id, async (c) => { ... })` to atomically:
   a. Update `c.financial_data` with corrections: `monthly_net_income` (parseFloat), `marital_status`
   b. Update `c.extended_financial_data` with: `berufsstatus` (income_source), `anzahl_unterhaltsberechtigte` (parseInt number_of_dependents)
   c. Update `c.aktuelle_pfaendung` = active_garnishments
   d. Write `c.second_letter_financial_snapshot` with ALL snapshot fields:
      - `monthly_net_income`: parseFloat(req.body.monthly_net_income)
      - `income_source`: req.body.income_source
      - `marital_status`: req.body.marital_status
      - `number_of_dependents`: parseInt(req.body.number_of_dependents)
      - `active_garnishments`: req.body.active_garnishments === true
      - `has_new_creditors`: req.body.has_new_creditors === true
      - `new_creditors`: req.body.has_new_creditors ? req.body.new_creditors.map(c => ({ name: c.name, amount: parseFloat(c.amount) })) : []
      - `submitted_at`: new Date()
   e. Set `c.second_letter_status = 'FORM_SUBMITTED'`
   f. Set `c.second_letter_form_submitted_at = new Date()`
   g. Return c
5. Log success: `console.log('[SecondLetterForm] Form submitted for client:', req.clientId)`
6. Return 200 `{ success: true, submitted_at: updatedClient.second_letter_form_submitted_at }`
7. Wrap entire handler in try/catch — return 500 on unexpected errors with console.error logging
