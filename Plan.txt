## 🎯 Systemübersicht

### Automatisierungsflow im Überblick

```
Agent erstellt Mandant → Portal-Link → Mandant lädt Dokumente hoch →
AI-Verarbeitung → Manuelle Prüfung (falls nötig) → Mandanten-Bestätigung →
Automatische Gläubiger-Anschreibung → Antworten verarbeiten → Schuldenregulierungsplan

```

**[BILD EINFÜGEN: Systemarchitektur-Diagramm]***Zeigt: Zendesk als zentrale Hub, Portal, AI-Pipeline, E-Mail-Automation*

### Ticket-Struktur pro Mandant

**4 Haupttickets werden automatisch erstellt:**

1. **Portal-Zugang** → Einmaliger Login-Link
2. **Gläubiger-Prozess** → Dokumentenprüfung + Bestätigung
3. **Gläubiger-Anschreibung** → Side Conversations für alle Gläubiger
4. **Schuldenregulierung** → Finale Pläne und Dokumente

## 👥 Agent-Workflows

### Workflow 1: Payment-Agent (Zahlungsüberwachung)

**Täglich prüfen:**

1. Dashboard: "Erste Raten eingegangen heute"
2. Für jeden Mandanten mit bezahlter Rate:
    - Zendesk öffnen
    - Mandant suchen (über Aktenzeichen)
    - **Neues Ticket erstellen:** "Gläubiger-Prozess starten: [NAME]"
    - **Zuweisen an:** Gläubiger-Team
    - **Priorität:** Normal
    - **Tags:** `phase-documents`, `payment-confirmed`

**[BILD EINFÜGEN: Payment-Agent Dashboard]***Zeigt: Übersicht der eingegangenen Zahlungen*

### Workflow 2: Gläubiger-Agent (Dokumentenverarbeitung)

**View nutzen: "⏳ Gläubiger-Prozess Tickets"**

**Für jedes Ticket:**

1. **Öffnen:** Ticket "Gläubiger-Prozess starten"
2. **Prüfen:** Wurden Dokumente hochgeladen?
    - **JA:** Automatische Verarbeitungsnotiz vorhanden → Weiter zu Schritt 3
    - **NEIN:** Macro "Dokumente anfordern" → Ticket auf Pending
3. **Manuelle Prüfung (falls nötig):**
    - Button "Dokumente manuell prüfen" klicken
    - Modal mit Dokumentenvorschau öffnet sich
    - Unklare Dokumente korrigieren
    - Bestätigen und schließen
4. **Gläubigerliste senden:**
    - Macro "Gläubigerliste zur Bestätigung" verwenden
    - Ticket Status → Pending
    - Warten auf Mandanten-Bestätigung
5. **Nach Bestätigung:**
    - Automatisches neues Ticket wird erstellt
    - System startet Gläubiger-Anschreibung
    - Original-Ticket kann geschlossen werden

### Workflow 3: Gläubiger-Monitoring Agent

**View nutzen: "📧 Gläubiger-Anfragen aktiv"**

**Täglich überwachen:**

1. **Offene Gläubiger-Anschreibung Tickets prüfen**
2. **Antworten verarbeiten:**
    - Side Conversations prüfen
    - AI-Extraktion kontrollieren
    - Bei Unklarheiten manuell korrigieren
3. **Timeout-Management:**
    - Nach 14 Tagen automatische Fallback-Beträge prüfen
    - Bei Bedarf manuelle Nachfass-E-Mails senden
4. **Abschluss-Prüfung:**
    - Wenn alle Gläubiger abgeschlossen
    - Button "Schuldenregulierungsplan erstellen"
    - Mandanten-Finanzdaten eingeben
    - Plan-Generierung starten

### Workflow 4: Plan-Agent (Schuldenregulierung)

**View nutzen: "✅ Bereit für Schuldenregulierung"**

**Für jedes bereite Ticket:**

1. **Finanzdaten erfassen:**
    - Modal mit Mandanten-Daten öffnen
    - Einkommen, Familienstand, etc. eingeben
    - Pfändbare Summe automatisch berechnen lassen
2. **Plan erstellen:**
    - Automatische Quotenberechnung prüfen
    - Plan-Generierung starten
    - 5-seitige PDFs für alle Gläubiger erstellen
3. **Dokumente verwalten:**
    - PDFs herunterladen und prüfen
    - An Gläubiger per E-Mail senden
    - Oder postalisch versenden
4. **Abschluss:**
    - Ticket als "Solved" markieren
    - Mandant Status → "Abgeschlossen"
    - Dokumentation vervollständigen

## 🔧 Troubleshooting

### Häufige Probleme und Lösungen

### Problem 1: Automatische Verarbeitung funktioniert nicht

**Symptome:**

- Mandant hat Dokumente hochgeladen
- Keine automatische Notiz im Ticket erscheint
- Webhook-Fehler

**Lösung:**

1. **Webhook-Status prüfen:**
    - Portal-Dashboard → Webhook-Logs
    - Fehlerhafte API-Calls identifizieren
2. **Manuelle Verarbeitung:**
    - Agent kann manuell "Dokumente neu verarbeiten" triggern
    - Internal Note mit Link zum Portal-Dashboard
3. **Ticket-Update:**
    
    ```
    ⚠️ AUTOMATISCHE VERARBEITUNG FEHLGESCHLAGEN
    
    Fehler: Webhook-Timeout nach 30 Sekunden
    Zeitpunkt: {{error_timestamp}}
    
    🔧 LÖSUNGSSCHRITTE:
    1. Dokumente wurden erfolgreich hochgeladen
    2. Manuelle Verarbeitung wurde gestartet
    3. Ergebnisse folgen in separater Notiz
    
    Agent: Bitte Portal-Dashboard prüfen für Details
    
    ```
    

### Problem 2: Gläubiger-E-Mail bounce zurück

**Symptome:**

- Side Conversation E-Mail nicht zustellbar
- Bounce-Nachricht erhalten
- Gläubiger kann nicht erreicht werden

**Lösung:**

1. **Alternative E-Mail suchen:**
    - Original-Dokument prüfen
    - Google/Firmen-Website checken
    - Telefon-Kontakt aufnehmen
2. **Ticket-Update:**
    
    ```
    📧 E-MAIL BOUNCE - Sparkasse Dresden
    
    ❌ Bounce-Grund: Mailbox nicht erreichbar
    Original E-Mail: forderungen@sparkasse-dresden.de
    
    🔍 NÄCHSTE SCHRITTE:
    1. Alternative E-Mail-Adresse recherchieren
    2. Telefonische Kontaktaufnahme
    3. Postalischer Versand als Fallback
    
    Agent: Bitte manuelle Nachbearbeitung
    
    ```
    
3. **Manuelle Bearbeitung:**
    - Agent recherchiert korrekte Kontaktdaten
    - Neue Side Conversation mit korrekter E-Mail
    - Dokumentation der Änderung

### Problem 3: AI-Extraktion ungenau

**Symptome:**

- Falsche Beträge erkannt
- Gläubiger nicht korrekt identifiziert
- Confidence-Score sehr niedrig (<60%)

**Lösung:**

1. **Sofortige manuelle Prüfung:**
    - Button "Dokumente manuell prüfen"
    - Alle Daten einzeln kontrollieren
    - Korrekturen vornehmen
2. **Dokumentation:**
    
    ```
    🤖 AI-EXTRAKTION KORRIGIERT
    
    Original AI-Ergebnis:
    • Gläubiger: "Unbekannt" (Confidence: 45%)
    • Betrag: 0,00 EUR (Confidence: 30%)
    
    Manuelle Korrektur:
    • Gläubiger: "Inkasso Müller GmbH"
    • Betrag: 1.200,00 EUR
    • Aktenzeichen: IM/456
    
    Agent: {{agent_name}}
    Korrigiert: {{correction_timestamp}}
    
    ```
    

### Problem 4: Mandant bestätigt Gläubigerliste nicht

**Symptome:**

- E-Mail zur Bestätigung versendet
- Keine Reaktion nach 7+ Tagen
- Gläubiger-Prozess blockiert

**Lösung:**

1. **Erinnerungs-E-Mail senden:**
    
    ```
    Macro: "Bestätigung Erinnerung"
    
    Betreff: ERINNERUNG: Bestätigung Ihrer Gläubigerliste
    
    Sehr geehrte/r {{requester.name}},
    
    vor {{days_since_sent}} Tagen haben wir Ihnen Ihre Gläubigerliste
    zur Bestätigung gesendet.
    
    ⚠️ WICHTIG: Ohne Ihre Bestätigung können wir nicht fortfahren.
    
    Bitte loggen Sie sich ein: {{portal_link}}
    
    Bei Problemen kontaktieren Sie uns: {{phone_number}}
    
    ```
    
2. **Telefonische Nachfassung:**
    - Agent ruft Mandanten an
    - Bestätigung telefonisch einholen
    - Interne Notiz mit Gesprächsprotokoll
3. **Eskalation:**
    - Nach 14 Tagen ohne Reaktion
    - Ticket an Supervisor weiterleiten
    - Weitere Maßnahmen prüfen
